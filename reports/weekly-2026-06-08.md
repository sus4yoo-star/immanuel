## 1. 아직 해결이 필요한 문제 🔴

### 1-1. 스마트폰 홈 화면 아이콘 오류 (지난 주에서 이어짐)

- **무슨 문제인가요?**
  홈 화면에 앱을 추가할 때 SVG 그림 형식만 사용합니다. 갤럭시 등 안드로이드 기기 상당수는 SVG를 앱 아이콘으로 지원하지 않아 깨진 아이콘이 표시됩니다. 또한 현재 '안전 영역이 확보된 아이콘(maskable icon)'으로도 SVG를 지정해 두어, 일부 기기에서 아이콘이 잘리거나 배경 없이 표시될 수 있습니다.
- **위험도**: 중간 — 첫인상과 직결되는 설치 경험 문제.
- **권장 조치**: 192×192픽셀, 512×512픽셀 PNG 이미지를 추가하고 manifest에 등록.
- **관련 파일**: `app/manifest.ts`

### 1-2. 분석·오류 추적 도구 미연동 (지난 주에서 이어짐)

- **무슨 문제인가요?**
  오류 안전망 화면(error.tsx)이 추가됐지만, 오류가 발생해도 내용이 어디에도 기록되지 않습니다. 사용자가 어디서 앱을 떠나는지, 어느 장면에서 문제가 생기는지 파악할 방법이 없습니다.
- **위험도**: 중간 — 운영 중인 서비스라면 '보이지 않는 문제'가 쌓일 수 있음.
- **권장 조치**: Sentry(무료 플랜 있음) 또는 Vercel Analytics 연동.
- **관련 파일**: `app/error.tsx` 7번 줄

## 2. 새로 발견한 문제 🟡

### 2-1. 가장 강력한 보안 설정(CSP) 누락

- **무슨 문제인가요?**
  지난 주 보안 헤더가 추가됐지만, 가장 중요한 **콘텐츠 보안 정책(Content-Security-Policy, CSP)** 은 포함되지 않았습니다. CSP는 외부의 악성 스크립트가 앱 안에서 실행되는 것을 막는 핵심 방어막입니다.
- **현재 위험도**: 낮음 — 외부 입력이 없고 정적 앱이라 즉각적 위협은 낮음. 그러나 '있어야 할 기본 설정'입니다.
- **관련 파일**: `next.config.ts`

### 2-2. 오류 안전망 화면에서 오류 내용 미기록

- **무슨 문제인가요?**
  새로 추가된 오류 화면(`app/error.tsx`)은 `error` 라는 오류 정보를 받을 수 있지만, 현재 코드에서는 그 내용을 아무것도 하지 않고 무시합니다. 빈 메모 칸을 만들어 두고 아무것도 적지 않은 상태입니다.
- **위험도**: 중간 — 운영 환경에서 어떤 오류가 발생했는지 전혀 알 수 없음.
- **관련 파일**: `app/error.tsx` 5번 줄

### 2-3. 공유 버튼 후 '복사됨' 표시가 자동으로 사라지지 않을 수 있음

- **무슨 문제인가요?**
  '묵상 노트 공유' 버튼을 누르면 잠시 "복사됨"으로 표시되다 1.5초 후 원래대로 돌아옵니다. 그런데 1.5초가 되기 전에 앱을 닫으면, 이미 닫힌 화면에 상태 변경을 시도해 불필요한 오류가 내부적으로 발생할 수 있습니다.
- **위험도**: 낮음 — 사용자가 직접 느끼는 문제는 아님.
- **관련 파일**: `components/GospelGame.tsx` 244번 줄

### 2-4. 서비스 워커 교체 시 사용자 알림 없음

- **무슨 문제인가요?**
  앱이 업데이트되면 브라우저 뒤에서 새 버전으로 교체됩니다(skipWaiting 설정). 이 교체가 자동으로 일어나는 동안 화면이 이전 버전과 새 버전이 섞여 보일 수 있습니다. "새 버전이 준비됐습니다. 새로고침하시겠습니까?" 같은 안내가 없어 드물게 이상하게 보이는 상황이 발생할 수 있습니다.
- **위험도**: 낮음 — 새로고침하면 해결됨.
- **관련 파일**: `public/sw.js`

### 2-5. 저장된 진행 상황 불완전한 검증

- **무슨 문제인가요?**
  이어서 읽기를 위해 브라우저 저장소에서 데이터를 불러올 때, '배열인가?'만 확인하고 배열 안의 항목들이 올바른 구조인지는 확인하지 않습니다. 드물게 저장소 데이터가 손상된 경우 앱이 예상치 못하게 작동할 수 있습니다.
- **위험도**: 낮음 — 실제 발생 가능성은 낮지만, 방어 코드 추가 권장.
- **관련 파일**: `components/GospelGame.tsx` 178번 줄

### 2-6. 성경 본문 서식 불일치

- **무슨 문제인가요?**
  누가복음 23장 43절 본문만 끝에 마침표(.)가 붙어 있고, 요한복음 19장 30절은 본문 중간에 작은따옴표(`'다 이루었다'`)를 쓰고 있습니다. 다른 구절들은 이 서식을 사용하지 않아 일관성이 없습니다. 개역개정 원문을 그대로 옮기는 것이 원칙이므로 원문 확인이 필요합니다.
- **위험도**: 낮음 — 표시에는 문제없지만, 정확성 차원에서 확인 필요.
- **관련 파일**: `components/GospelGame.tsx` 15~16번 줄

## 3. 앞으로를 위한 개선 제안 🟠

### 3-1. 콘텐츠·코드 파일 분리 (유지보수성)

- **무슨 문제인가요?**
  현재 60개 이상의 장면 대본, 11개의 성경 구절, 앱 화면 코드가 모두 하나의 파일(GospelGame.tsx, 392줄)에 들어 있습니다. 앞으로 장면을 추가하거나 성경 구절을 수정할 때 개발자가 아닌 분이 내용을 편집하기 어렵고 실수가 생길 수 있습니다.
- **제안**: 장면 대본과 성경 구절을 별도 JSON 파일로 분리.

### 3-2. 오프라인 사용 범위 확대

- **무슨 문제인가요?**
  현재 오프라인 설치 시 첫 페이지(`/`)만 캐시에 저장됩니다. 앱 화면을 구성하는 JavaScript·CSS 파일들은 처음 접속 후에야 캐시됩니다. 설치 직후 인터넷 없이 앱을 열면 일부 파일이 없어 정상 작동하지 않을 수 있습니다.
- **관련 파일**: `public/sw.js` 4번 줄

### 3-3. 장(chapter) 단위 이동 기능

- **무슨 문제인가요?**
  이야기를 다시 볼 때 원하는 막(예: "3막 십자가")으로 바로 이동하는 방법이 없습니다. 읽던 위치 저장 기능이 생겼지만, 특정 장면을 다시 읽고 싶을 때는 처음부터 순서대로 눌러야 합니다.
- **체감 영향**: 재방문 사용자의 활용도와 만족도 향상.

## 6. 잘 된 점 ✅

- 지난 주 리포트의 권고 사항 7가지를 단 3일 만에 모두 반영한 점이 돋보입니다.
- '이어서 읽기' 기능이 자연스럽게 구현되어, 첫 화면에서 읽던 장(chapter)을 안내해 줍니다.
- 오류 안전망 화면이 앱의 디자인 톤(금색·남색)을 유지하며 한국어로 친절하게 안내합니다.
- 보안 헤더 4종이 추가되어 기본 웹 보안 기준을 갖추었습니다.
- 서비스 워커가 '네트워크 우선' 전략으로 전환되어, 새 콘텐츠를 배포하면 사용자에게 더 빠르게 반영됩니다.
