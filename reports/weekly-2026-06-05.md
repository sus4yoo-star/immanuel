## 한 줄 요약

앱 전체 흐름은 잘 작동하지만, **스마트폰 홈 화면 아이콘 설치**, **업데이트 후 이전 화면 표시 문제**, **접근성** 세 가지를 빠른 시일 내에 조치하면 사용 경험이 크게 향상됩니다.

## 1. 버그 · 오작동 위험 🔴

### 1-1. 앱 업데이트 후 이전 내용이 계속 보이는 문제 (중요)

- **무슨 문제인가요?**
  앱을 업데이트해도 이전 버전의 화면이 계속 보일 수 있습니다. 스마트폰이나 컴퓨터가 이전 내용을 저장(캐시)해 두는데, 그 저장 이름이 항상 `immanuel-v1`으로 고정돼 있기 때문입니다.
- **언제 문제가 되나요?**
  새로운 성경 구절이나 장면을 추가한 뒤 배포했는데, 사용자에게는 여전히 이전 버전이 표시될 수 있습니다.
- **위험도**: 높음 — 새 내용을 배포해도 사용자가 즉시 볼 수 없습니다.
- **관련 파일**: `public/sw.js` 1번 줄

### 1-2. 잘못된 장면 이름으로 인한 앱 전체 멈춤 위험 (중요)

- **무슨 문제인가요?**
  현재 60개 이상의 장면이 서로 연결돼 있는데, 어느 한 장면의 '다음 장면 이름'이 잘못 입력되면 앱이 흰 화면과 함께 완전히 멈춥니다. 오류를 잡아주는 안전장치가 없습니다.
- **위험도**: 중간 — 장면이 많아질수록 오타 발생 가능성이 커집니다.
- **관련 파일**: `components/GospelGame.tsx` 129번 줄

### 1-3. 스마트폰 앱 아이콘 오류 (중간)

- **무슨 문제인가요?**
  홈 화면에 앱을 추가할 때 SVG 형식 이미지만 사용합니다. 안드로이드 등 일부 기기는 SVG를 앱 아이콘으로 지원하지 않아 깨진 아이콘이나 기본 아이콘이 보일 수 있습니다.
- **위험도**: 중간 — PWA(홈 화면 앱) 설치 경험에 직접 영향.
- **관련 파일**: `app/manifest.ts`

## 2. 미완성 기능 🟡

### 2-1. 읽던 위치 저장 기능 없음

- **무슨 문제인가요?**
  앱을 닫으면 진행 상황이 사라지고 처음부터 다시 시작해야 합니다. 5막으로 구성된 긴 이야기를 이어서 볼 수 없습니다.
- **체감 영향**: 이야기를 끝까지 읽지 못하는 사용자가 많아질 수 있습니다.

### 2-2. 묵상 노트 내보내기/공유 없음

- **무슨 문제인가요?**
  이야기 중간중간 마음에 새긴 말씀이 '묵상 노트'에 쌓이지만, 이를 카톡이나 메모로 내보내는 기능이 없습니다.
- **체감 영향**: 공동체 나눔이나 개인 기록 용도로 확장하기 어렵습니다.

### 2-3. 개역개정 성경 본문 입력 도구 (개발용)

- **현재 상태**: 성경 본문을 입력하는 편집 화면이 있지만, 개발 환경에서만 표시되며 앱 배포 시에는 숨겨집니다. 이미 주요 구절은 모두 입력돼 있으므로 현재로선 큰 문제가 아닙니다.

## 3. 개선이 필요한 부분 🟠

### 3-1. 글자 크기 조절 차단 (접근성 문제)

- **무슨 문제인가요?**
  현재 설정상 사용자가 화면을 확대하거나 글자 크기를 키울 수 없습니다. 시력이 좋지 않은 분들이 앱을 사용하기 어렵습니다.
- **관련 국제 기준**: 웹 접근성 지침(WCAG)에서 사용자 확대를 막지 말도록 권고합니다.
- **위험도**: 중간 — 법적·윤리적 접근성 이슈.
- **관련 파일**: `app/layout.tsx` 16–17번 줄

### 3-2. 앱 성능 최적화 미적용

- **무슨 문제인가요?**
  `next.config.ts` 파일이 사실상 비어 있어 이미지 최적화, 보안 헤더, 압축 등 기본적인 성능·보안 설정이 하나도 없습니다.

### 3-3. 오류 추적·분석 도구 없음

- **무슨 문제인가요?**
  실제 사용자가 앱 이용 중 오류를 만나도 개발팀이 알 방법이 없습니다. 얼마나 많은 사람이 어느 장면까지 읽는지도 파악할 수 없습니다.

## 4. 보안 점검 🛡️

### 4-1. 보안 헤더 없음 (낮은 위험)

- 웹 표준 보안 설정(CSP, X-Frame-Options 등)이 적용되지 않았습니다. 콘텐츠 특성상 즉각적인 위협은 낮지만, 정식 서비스라면 추가하는 것이 좋습니다.

### 4-2. 로컬 저장소 사용 (개발 환경 한정, 낮은 위험)

- 성경 본문 편집 기능은 개발 환경에서만 기기 내 저장소(localStorage)를 사용합니다. 민감한 개인정보는 저장하지 않으므로 현재 위험도는 낮습니다.

### 4-3. 서비스 워커 오류 기록 없음 (낮은 위험)

- `RegisterSW.tsx`가 오류를 소리 없이 무시합니다. 서비스 워커 등록 실패 시 아무 알림이 없어 오프라인 기능 장애를 인지하기 어렵습니다.

## 6. 잘 된 점 ✅

- 5막 구성의 이야기 흐름이 논리적으로 잘 연결돼 있습니다.
- 선택에 따른 분기가 있지만 최종적으로는 모든 길이 이어지도록 설계돼 있어 사용자가 중요한 장면을 놓치지 않습니다.
- 개역개정 본문과 창작 서술을 명확히 구분하는 설계가 돋보입니다.
- PWA 기본 구조(서비스 워커 + 매니페스트)를 갖추어 오프라인에서도 동작합니다.
- 개발 환경과 운영 환경을 `process.env.NODE_ENV`로 분리해 편집 도구가 실제 사용자에게 노출되지 않습니다.
